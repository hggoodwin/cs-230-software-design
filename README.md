# cs-230-software-design
Software Design Document for The Gaming Room – CS 230
Hunter Goodwin
Client Summary
The client, The Gaming Room, requested the design of a multi-user game application called Draw It or Lose It. The application needed to support multiple teams, rounds, and concurrent game sessions while rendering images progressively as clues. The system had to be scalable and capable of running on multiple platforms beyond Android.

What I Did Well
I did particularly well in organizing system requirements clearly and breaking down architectural components in a way that would be understandable to both technical and non-technical stakeholders. The document clearly communicates system constraints, platform considerations, and development strategies.

How the Design Document Helped
Working through the design document helped me think about system architecture before implementation. It forced me to consider scalability, operating system differences, memory management, and server considerations before writing code. This planning phase reduces future development errors.

What I Would Revise
If I were to revise one section, I would expand further on deployment strategy and cloud hosting options. Providing more detailed infrastructure diagrams would strengthen the documentation.

Interpreting User Needs
I interpreted the client’s needs by identifying functional requirements such as multiple team support and timed rounds, as well as non-functional requirements like scalability and cross-platform compatibility. Considering user needs is critical because software must solve the user’s real problem, not just demonstrate technical ability.

My Design Approach
I approached the design by breaking the system into logical components and evaluating platform trade-offs. In the future, I would continue using structured documentation, UML diagrams, and early architecture planning to guide development.
